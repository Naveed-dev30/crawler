> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Read-only admin dashboard page listing all mobile-chat threads with a slide-over showing assignment/escalation history and the conversation.

**Architecture:** Server-rendered Blade list (20/page) + AJAX offcanvas detail partial, mirroring the Bid Insights page and the Bids-page detail panel. New `ChatController` with `index`/`detail`, both in the existing `admin` route group. No writes anywhere.

**Tech Stack:** Laravel 10, Blade, Bootstrap 5 (Sneat theme), boxicons, PHPUnit feature tests (sqlite :memory:).

```php
    public function index(Request $request)
    {
        $search = trim((string) $request->query('search', ''));
        $status = $request->query('status', '');

        $threads = Thread::query()
            ->with(['assignedUser', 'proposal'])
            ->withCount([
                'messages',
                'logs as escalations_count' => fn ($q) => $q->where('type', 'escalation'),
            ])
            ->when($search !== '', function ($q) use ($search) {
                $q->where(function ($q) use ($search) {
                    $q->where('project_id', 'like', "%{$search}%")
                        ->orWhereHas('proposal', fn ($p) => $p->where('title', 'like', "%{$search}%"))
                        ->orWhereHas('assignedUser', fn ($u) => $u->where('name', 'like', "%{$search}%"));
                });
            })
            ->when(in_array($status, ['fresh', 'replied'], true), fn ($q) => $q->where('status', $status))
            ->when($status === 'blocked', fn ($q) => $q->where('blocked', true))
            ->orderByRaw('last_client_message_at IS NULL')
            ->orderByDesc('last_client_message_at')
            ->paginate(20)
            ->withQueryString();

        return view('content.pages.chats', ['threads' => $threads]);
    }
```

```php
    public function detail(Thread $thread)
    {
        $thread->load([
            'assignedUser',
            'proposal',
            'messages' => fn ($q) => $q->orderBy('message_time'),
            'messages.sender',
            'messages.attachments',
            'logs' => fn ($q) => $q->orderBy('created_at'),
            'logs.fromUser',
            'logs.toUser',
        ]);

        // AI matches are never logged; reconstruct the first assignee.
        $firstAssignee = $thread->logs->first()?->fromUser ?? $thread->assignedUser;

        return view('_partials.chat-thread-detail', [
            'thread' => $thread,
            'firstAssignee' => $firstAssignee,
        ]);
    }
```
